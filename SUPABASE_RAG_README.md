# Supabase RAG System

This project provides a Retrieval-Augmented Generation (RAG) system using Supabase as the vector database. It allows you to:

1. Insert documents into a Supabase table with vector embeddings
2. Perform semantic searches to find the most relevant documents

## Prerequisites

- Python 3.7+
- A Supabase account and project
- Supabase table with vector extension enabled

## Installation

1. Install the required Python packages:

```bash
pip install supabase sentence-transformers python-dotenv numpy
```

2. Set up your Supabase project:

   - Create a new Supabase project if you don't have one
   - Enable the `pgvector` extension in your Supabase SQL editor:
     ```sql
     create extension if not exists vector;
     ```

3. Create a table for your documents:

```sql
create table documents (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  titulo text,
  documento text,
  embedding vector(384)
);

-- Create an index for faster vector searches
create index on documents using ivfflat (embedding vector_cosine_ops) with (lists = 100);
```

Note: The embedding dimension (384) is based on the default "all-MiniLM-L6-v2" model. If you use a different model, adjust the dimension accordingly.

## Configuration

1. Create a `.env` file in your project directory with your Supabase credentials:

```
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_KEY=your-supabase-api-key
```

## Usage

### Basic Usage

The `supabase_rag.py` file contains the `SupabaseRAG` class that provides methods for:

- Inserting documents with embeddings
- Searching for relevant documents

### Example Script

The `example_supabase_rag.py` script demonstrates how to:

1. Connect to your Supabase project
2. Create the necessary database function for vector search
3. Insert example documents
4. Perform semantic searches

To run the example:

```bash
python example_supabase_rag.py
```

### Creating the Match Function

Before using the search functionality, you need to create a stored procedure in Supabase. The `create_match_documents_function()` method will create this for you, or you can run the following SQL in your Supabase SQL editor:

```sql
create or replace function match_documents (
  query_embedding vector,
  match_threshold float,
  match_count int
)
returns table (
  id bigint,
  titulo text,
  documento text,
  similarity float
)
language plpgsql
as $$
begin
  return query
  select
    id,
    titulo,
    documento,
    1 - (embedding <=> query_embedding) as similarity
  from documents
  where 1 - (embedding <=> query_embedding) > match_threshold
  order by similarity desc
  limit match_count;
end;
$$;
```

## Custom Implementation

To use this system in your own project:

1. Import the `SupabaseRAG` class:

```python
from supabase_rag import SupabaseRAG
```

2. Initialize with your Supabase credentials:

```python
rag = SupabaseRAG(supabase_url, supabase_key)
```

3. Insert documents:

```python
rag.insert_document("Document Title", "Document text content...")
```

4. Search for relevant documents:

```python
results = rag.search_documents("Your search query", top_k=5)
```

## Customization

You can customize the embedding model by changing the `model_name` parameter in the `get_embedding` function. The default model is "all-MiniLM-L6-v2", which provides a good balance between performance and quality.

If you change the model, make sure to update the vector dimension in your Supabase table schema accordingly.
