Como Este App Funciona (Alimentado por Pixeltable)

Este agente de IA multimodal é construído usando Pixeltable, a infraestrutura de dados de IA de código aberto. Pixeltable fornece blocos de construção essenciais para simplificar aplicativos de IA complexos:

  • Tabelas: Armazenar referências a dados multimodais de forma eficiente.
  • Colunas Computadas: Executar automaticamente funções Python ou modelos de IA quando os dados mudam.
  • Views & Iterators: Transformar dados on-the-fly sem duplicação.
  • Embedding Indexes: Habilitar pesquisa semântica rápida.
  • Query Functions: Definir lógica de pesquisa reutilizável.

Aqui está como esses conceitos impulsionam os recursos que você vê, com trechos de código reais do `setup_pixeltable.py` desta aplicação:

1. Gerenciando seus Arquivos (Tabelas & Colunas Computadas)

Arquivos são gerenciados usando Pixeltable Tabelas, que armazenam referências eficientemente usando tipos multimodais integrados:


# From setup_pixeltable.py
images = pxt.create_table(
    "agents.images",
    {"image": pxt.Image, "uuid": pxt.String, "timestamp": pxt.Timestamp},
    if_exists="ignore",
)
documents = pxt.create_table(
    "agents.collection",
    {"document": pxt.Document, "uuid": pxt.String, "timestamp": pxt.Timestamp},
    if_exists="ignore",
)

Processing happens via Colunas Computadas, que aplicam automaticamente funções (como redimensionamento ou chamadas de API) quando os dados são adicionados:


# From setup_pixeltable.py
# Gerar miniaturas automaticamente quando imagens são adicionadas
images.add_computed_column(
    thumbnail=pxt_image.b64_encode(
        pxt_image.resize(images.image, size=(96, 96)) # Sidebar size
    ),
    if_exists="ignore",
)

# Extract audio automatically when videos are added
videos.add_computed_column(
    audio=extract_audio(videos.video, format="mp3"),
    if_exists="ignore"
)

2. Respondendo Perguntas (Fluxo de Trabalho, Views, Índices, Consultas)

A lógica central do agente funciona como um fluxo de trabalho declarativo na tabela agents.tools. Os passos principais envolvem transformar dados e realizar pesquisas semânticas:

  1. Transformação de Dados (Views & Iterators): Os documentos são fragmentados on-the-fly usando uma View e um Iterator DocumentSplitter, evitando duplicação de dados: 
    
# From setup_pixeltable.py
chunks = pxt.create_view(
    "agents.chunks",
    documents, # Base table
    iterator=DocumentSplitter.create(
        document=documents.document,
        separators="token_limit", # Split by token count
        limit=300
    ),
    if_exists="ignore",
)
  2. Pesquisa Semântica (Índices de Embedding): Os índices são adicionados às tabelas/views para habilitar uma pesquisa de similaridade rápida usando modelos integrados como Sentence Transformers ou CLIP: 
    
# From setup_pixeltable.py
# Indexar fragmentos de documentos para pesquisa de texto
chunks.add_embedding_index(
    "text",
    string_embed=sentence_transformer.using(model_id=config.EMBEDDING_MODEL_ID),
    if_exists="ignore",
)
# Indexar imagens para pesquisa multimodal
images.add_embedding_index(
    "image",
    embedding=clip.using(model_id=config.CLIP_MODEL_ID),
    if_exists="ignore",
)
  3. Recuperação de Contexto (Funções de Consulta): A lógica de pesquisa reutilizável é definida usando o decorador `@pxt.query`. Essas funções são chamadas por colunas computadas no fluxo de trabalho principal: 
    
# From setup_pixeltable.py
@pxt.query
def search_documents(query_text: str):
    sim = chunks.text.similarity(query_text)
    return (
        chunks.order_by(sim, asc=False)
        .select(chunks.text, source_doc=chunks.document, sim=sim)
        .where(sim > 0.8)
        .limit(10)
    )
  4. Orquestração de Fluxo de Trabalho: A sequência de colunas computadas na tabela agents.tools define todo o processo declarativamente. Ele inclui:
    • Chamando funções de pesquisa (como search_documents mostrado acima).
    • Usando Funções Definidas pelo Usuário (UDFs) para executar código Python personalizado, como buscar dados externos. UDFs são criados com o decorador @pxt.udf: 
      
# From functions.py
@pxt.udf
def get_latest_news(topic: str) -> str:
    """Fetch latest news for a given topic using NewsAPI."""
    # ... implementation omitted for brevity ...
    return formatted_news
    • Habilitando Chamada de Ferramentas registrando UDFs como ferramentas para o LLM. Pixeltable fornece helpers para isso: 
      
# From setup_pixeltable.py
# Define available tools
tools = pxt.tools(
    # functions.get_latest_news,
    # functions.fetch_financial_data,
    # functions.search_news
)

# Pass tools to the LLM (Claude) via a computed column
tool_agent.add_computed_column(
    initial_response=messages(
        model=config.CLAUDE_MODEL_ID,
        system=tool_agent.initial_system_prompt,
        messages=[{"role": "user", "content": tool_agent.prompt}],
        tools=tools, # Make tools available
        tool_choice=tools.choice(required=True), # Force tool use
        # ... other parameters ...
    ),
    if_exists="replace",
)

# Execute the chosen tool using invoke_tools
tool_agent.add_computed_column(
    tool_output=invoke_tools(tools, tool_agent.initial_response),
    if_exists="replace"
)
    • Chamando LLMs (como Claude ou Mistral) para raciocinar, gerar texto ou escolher ferramentas.
    Pixeltable gerencia automaticamente a ordem de execução e o armazenamento em cache com base nesses passos declarativos.

3. Gerando Imagens (Colunas Computadas & APIs)

A geração de imagens é acionada adicionando um prompt à tabela agents.image_generation_tasks. Uma Coluna Computada chama a API DALL-E 3 via a integração OpenAI do Pixeltable: 


# From setup_pixeltable.py
image_gen_tasks.add_computed_column(
    generated_image=openai.image_generations(
        prompt=image_gen_tasks.prompt,
        model=config.DALLE_MODEL_ID,
        size="1024x1024"
    ),
    if_exists="ignore",
)

4. Lembrando Coisas (Tabelas & Índices)

A persistência é alcançada usando Pixeltable Tabelas:

  • Banco de Memória: Itens salvos são armazenados em agents.memory_bank. Um Índice de Embedding habilita pesquisa semântica: 
    
# From setup_pixeltable.py
memory_bank.add_embedding_index(
    column="content",
    string_embed=sentence_embed_model, # Defined earlier
    if_exists="ignore",
)
  • Histórico de Chat: Turnos de conversa são registrados em agents.chat_history, permitindo recuperação de histórico recente ou pesquisa semântica sobre toda a conversa (também usando um índice de embedding).

By combining these primitives – Tabelas, Colunas Computadas, Views, e Índices de Embedding – Pixeltable fornece uma maneira poderosa e declarativa de construir e orquestrar aplicações de IA complexas e multimodais como este agente.
